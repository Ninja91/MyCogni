DESIGN RESPONSE

MyCogni preserves the denominator: not checked, ambiguous, blocked, and inconclusive remain visible. A broker's “completed” response is never silently promoted to verified removal.

01

LOCAL CUSTODY

Your identity profile is not another SaaS dossier.

Sensitive attributes stay encrypted on infrastructure the user controls. Each profile has an independently random wrapped key, making per-profile cryptographic deletion possible without keeping a reproducible master-derived key.

  • Encrypted state and evidence
  • External wrapping key
  • Explicit backup-expiry semantics

06 Delivery path

Evidence-gated, not calendar-gated

A 24-week route
to a narrow local v1.

The week ranges show order and size. Passing safety, legal, usability, and evidence gates—not elapsed time—unlocks the next release.

WEEKS 0–2

Build the foundation that can say “no.”

Land the P0 decisions in code contracts, stand up a synthetic broker simulator, and create an authenticated control-plane skeleton without real PII or traffic.

EXIT GATES
  • P0 ADRs accepted
  • Synthetic-only test path
  • Threat-to-requirement traceability

Optional local-assist evaluation comes after v1 and only ships if it reduces review time without changing safety outcomes or leaking PII.

07 Adversarial review

Five independent critique tracks

The design was attacked
before code was trusted.

These were AI-assisted role reviews—not claims of human security certification. Each track reviewed the same snapshot independently before the principal-role synthesis.

ML engineeringKeep intelligence out of authority.+

No model in v1. A future local runtime receives only sanitized bounded tasks and returns a schema-validated UntrustedSuggestion with supporting spans.
