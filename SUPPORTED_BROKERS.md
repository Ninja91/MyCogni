# Supported broker capabilities

There are no supported real-broker connectors yet.

The repository contains one synthetic registry example for schema and contract-test design. It does not contact a real organization and is not counted as coverage.

Future generated entries will disclose, per capability:

| Field | Meaning |
| --- | --- |
| Broker and category | named organization and public/private classification |
| Capability | `observe`, `prepare`, `submit`, `poll`, or `verify` |
| Maturity | `experimental`, `observe-tested`, `submission-candidate`, `trusted`, `quarantined`, or `retired` |
| Last validated / expires | freshness and automatic demotion boundary |
| Jurisdiction/policy | voluntary procedure or sourced state-law path |
| Required disclosure | exact identity attribute categories and reason |
| Human steps | CAPTCHA, MFA, identity document, login, or other intervention |
| Evidence method | receipt, assertion, single absence, corroborated absence, or unavailable |
| Recent quality | synthetic/canary outcome and age, without user PII |

MyCogni will not publish one undifferentiated broker count. A broker with only metadata is not equivalent to a fresh, independently verified automatic connector.
